# PPO 交易模型超參數調優手冊

本表整理了 PPO 核心參數的物理意義，以及針對交易環境調整數值時的預期影響。

| 參數名稱 | 意義 (Meaning) | 數值增加 ($+$) 的影響 | 數值減少 ($-$) 的影響 |
| :--- | :--- | :--- | :--- |
| **Learning Rate** | [cite_start]模型學習的步幅大小。目前建議從 $3 \times 10^{-4}$ 降至更低 。 | [cite_start]**訓練速度變快**；但過大會導致 `approx_kl` 爆炸，模型容易崩潰 。 | **訓練更穩定**；但過小會導致學得極慢，容易卡在死路。 |
| **N step** | 每次更新前收集的環境步數。 | **優勢估計更準確**；能看更多樣本才做決定，減少隨機雜訊干擾。 | **更新頻率變快**；計算負擔小，但估計偏差（Bias）會變大。 |
| **Batch size** | 每次梯度下降使用的樣本量。 | [cite_start]**梯度更新更穩定**；對於雜訊大的交易數據能提供更平滑的更新 。 | **計算速度快**；但更新方向會比較抖動，容易受雜訊影響。 |
| **N epochs** | 每次採樣後重複學習的次數。 | **提高採樣效率**；充分壓榨數據價值，但過多可能導致過擬合或策略跳變。 | **更新更保守**；能有效控制策略變動的幅度，增加穩定性。 |
| **Gamma ($\gamma$)** | 獎勵的折現因子。決定模型有多看重「未來」的回報。 | **更看重長期利益**；模型會傾向於考慮更長線的交易布局。 | **更看重短期即時回饋**；模型會變得比較「短視」，追求眼前獲利。 |
| **GAE lambda** | 優勢估計的偏差與方差平衡參數。 | **降低估計偏差**；但會增加方差（Variance），讓更新變抖。 | **降低方差**；讓訓練過程更平滑，但可能帶有系統性偏差。 |
| **Clip range** | 限制新舊策略變動幅度的閾值。 | **允許策略大幅改進**；訓練較快，但容易因一次異常行情學壞。 | **更新更嚴謹**；確保策略在「信任區域」內緩步提升。 |
| **Clip range vf** | 限制價值函數（Critic）更新幅度的閾值。 | **價值預測變動大**；能更快跟上獎勵變化，但預測穩定性較差。 | **價值預測較平穩**；避免價值函數因單次劇烈波動而大幅震盪。 |
| **Ent coef** | [cite_start]鼓勵模型探索的權重。防止策略過早收斂 [cite: 3, 4]。 | **增加探索性**；強制模型嘗試新行為，解決「不愛動」或「腦死」問題。 | [cite_start]**加速收斂**；但也容易發生「策略塌陷」，導致熵值歸零 [cite: 3, 4]。 |
| **Vf coef** | 價值函數損失在總損失中的比例。 | [cite_start]**更重視預測準確度**；有助於修正 `explained_variance` 為負的問題 [cite: 1, 2, 3]。 | **更重視獲利行為**；過低會讓模型只想賺錢卻不理解原因。 |
| **Max grad norm** | 梯度裁剪的閾值，防止梯度爆炸。 | **允許較大的權重修正**；適合變動劇烈的環境。 | **強制平滑梯度**；防止單次大賠（Outlier）導致模型權重壞掉。 |
| **Policy Net 第一層** | 策略網絡的第一層神經元數量。 | **特徵提取能力增強**；能處理更複雜的輸入特徵組合。 | **計算開銷變小**；模型較輕巧，不容易過擬合。 |
| **Policy Net 第二層** | 策略網絡的第二層神經元數量。 | **決策邏輯更細膩**；能學習更精密的進出場條件。 | **推理速度變快**；避免學習到過於複雜的偽規律。 |
| **Value Network** | 價值預測網絡的規模（Critic）。 | **評估狀態的能力增強**；有助於提供更高質量的優勢信號（Advantage）。 | **計算負荷降低**；適合狀態空間較為單純的任務。 |
| **Episode length** | 單次回合的最大交易步數。 | [cite_start]**觀察期更長**；模型能體驗長線持倉的結果。若為 0 代表環境有 Bug 。 | **學習更頻繁**；模型能更快重置，但可能漏掉長線趨勢邏輯。 |
| **Log interval** | 顯示訓練指標（如 Loss, Reward）的頻率。 | **減少日誌刷屏**；讓顯示界面看起來更乾淨。 | **監控更即時**；能讓你更早發現模型是否已經「練廢了」。 |
| **Save freq** | 模型自動存檔（Checkpoint）的步數間隔。 | **節省硬碟空間**；但如果崩潰，能找回的最近進度會比較遠。 | **容錯率更高**；崩潰時能快速找回最近的進度重新訓練。 |
| **Eval freq** | 暫停訓練並進行獨立回測/評估的頻率。 | **訓練更流暢**；節省切換到評估模式的時間成本。 | **及時止損**；讓你更早確認模型是否真的在獲利，而不是在空轉。 |
